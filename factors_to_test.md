# 需要预测与验证的因子

这个文档用于记录 `Store Sales - Time Series Forecasting` 当前已经确认会影响 leaderboard 分数的因子，以及下一步应优先测试的候选因子。

## 已验证因子

### 1. 最终融合权重

- 对象: `LightGBM / XGBoost` 最终加权比例
- 证据:
  - `version6`: `0.38022`
  - `version6.1` (`0.60 / 0.40`): `0.38041`
  - `version6.2` (`0.45 / 0.55`): `0.38017`
  - `version6.3` (`0.40 / 0.60`): `0.38015`
  - `version6.4` (`0.35 / 0.65`): `0.38016`
  - `version6.5` (`0.42 / 0.58`): `0.38016`
- 结论: 最终融合权重会稳定影响分数，当前最佳区间在 `0.40 / 0.60` 附近

### 2. 是否保留 `lag 730`

- 对象: 超长周期滞后特征 `730`
- 证据:
  - `version6.3`（保留 `730`）: `0.38015`
  - `version7.1`（去掉 `730`）: `0.38138`
- 结论: `lag 730` 提供了有效的长周期信息，移除后分数明显变差，应保留

## 下一步优先测试的候选因子

### 1. 是否保留 `lag 365`

- 原因: `730` 已确认有效，但 `365` 是否同样必要还没有被单独验证
- 建议实验:
  - `version7.2`: 保持 `version6.3` 全部配置不变，只移除 `365`
- 目标: 判断一年周期特征是否是另一个有效因子

### 2. 节假日特征筛选

- 原因: 当前 `selected_holidays` 仍可能包含噪声节假日
- 建议实验:
  - 只保留高影响节假日
  - 或按 `locale_name / transferred / type` 进一步筛选
- 目标: 判断节假日集合是否能继续降分

### 3. 过去协变量组合

- 对象: `transactions`、`oil`、`onpromotion`
- 原因: 这些变量都在主线方案中启用，但尚未做单独消融
- 建议实验:
  - 分别移除单个 past covariate
  - 保持模型和融合权重完全不变
- 目标: 找出真正有贡献的外部时间序列特征

### 4. 模型内 `lags` 组合

- 对象: 各组 `LightGBMModel` / `XGBModel` 的 `lags` 配置
- 原因: 当前是多组 `lags` 并行集成，但每组的边际贡献还不清楚
- 建议实验:
  - 删掉一组短周期 `lags`
  - 删掉一组长周期 `lags`
  - 对比是否有冗余分支
- 目标: 判断当前重型集成里是否存在可以裁剪的分支

### 5. `LightGBM` 与 `XGBoost` 参数

- 原因: 目前已经确认“模型间权重”重要，但“模型内参数”还没系统测
- 建议实验:
  - 优先微调 `learning_rate`、`max_depth`、`n_estimators`
  - 一次只改一个方向，避免多因子混杂
- 目标: 判断提升空间来自融合，还是也来自单模型质量

## 当前实验原则

- 所有新版本尽量基于 `version6.3`
- 每次只改一个因子
- 每次保留提交用 `csv`
- 记录版本号、改动点、Kaggle public score、结论

## 当前推荐顺序

1. 先验证 `lag 365`
2. 再做节假日筛选
3. 再做 past covariates 消融
4. 最后再考虑模型参数微调
