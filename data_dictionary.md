# Store Sales - Time Series Forecasting 字段说明

本文档用于说明当前目录下各个 CSV 文件的用途，以及每个字段的含义。

## 1. `train.csv`

用途：训练集，用来训练模型。每一行表示某一天、某一家门店、某一个商品大类在该组合下的实际销量和促销信息。

| 字段名 | 类型 | 含义 |
| --- | --- | --- |
| `id` | 整数 | 行唯一标识。在训练中通常不作为特征使用。 |
| `date` | 日期 | 记录对应的日期。 |
| `store_nbr` | 整数 | 门店编号。可与 `stores.csv` 关联。 |
| `family` | 字符串 | 商品大类，而不是单个具体商品。 |
| `sales` | 浮点数 | 目标变量，表示该日期、该门店、该商品大类的销量。 |
| `onpromotion` | 整数 | 该日期下，该门店该商品大类中正在促销的商品数量。 |

## 2. `test.csv`

用途：测试集，用来生成提交结果。结构与 `train.csv` 接近，但没有 `sales`。

| 字段名 | 类型 | 含义 |
| --- | --- | --- |
| `id` | 整数 | 测试集每一行的唯一标识。 |
| `date` | 日期 | 需要预测销量的日期。 |
| `store_nbr` | 整数 | 门店编号。 |
| `family` | 字符串 | 商品大类。 |
| `onpromotion` | 整数 | 该日期下，该门店该商品大类中处于促销状态的商品数量。 |

## 3. `sample_submission.csv`

用途：提交文件模板。

| 字段名 | 类型 | 含义 |
| --- | --- | --- |
| `id` | 整数 | 对应 `test.csv` 中每一行的唯一标识。 |
| `sales` | 浮点数 | 需要提交的预测销量。 |

## 4. `stores.csv`

用途：门店信息表，提供每个门店的静态属性。

| 字段名 | 类型 | 含义 |
| --- | --- | --- |
| `store_nbr` | 整数 | 门店编号，主键，可与主表按该字段关联。 |
| `city` | 字符串 | 门店所在城市。 |
| `state` | 字符串 | 门店所在州/省。 |
| `type` | 字符串 | 门店类型，通常是类别编码，如 `A`、`B`、`C`、`D`、`E`。 |
| `cluster` | 整数 | 门店聚类编号，表示该店所属的业务分群。 |

## 5. `transactions.csv`

用途：门店每日交易笔数表。每一行表示某家门店在某一天的总交易次数。

| 字段名 | 类型 | 含义 |
| --- | --- | --- |
| `date` | 日期 | 记录日期。 |
| `store_nbr` | 整数 | 门店编号。 |
| `transactions` | 整数 | 该门店该日的交易笔数。 |

## 6. `oil.csv`

用途：原油价格时间序列表。

| 字段名 | 类型 | 含义 |
| --- | --- | --- |
| `date` | 日期 | 记录日期。 |
| `dcoilwtico` | 浮点数 | WTI 原油价格。通常作为外部经济变量使用。 |

## 7. `holidays_events.csv`

用途：节假日和事件信息表。

| 字段名 | 类型 | 含义 |
| --- | --- | --- |
| `date` | 日期 | 节假日或事件发生的日期。 |
| `type` | 字符串 | 日期类型，例如 `Holiday`、`Event`、`Additional`、`Bridge`、`Transfer`、`Work Day`。 |
| `locale` | 字符串 | 作用范围，表示该节假日或事件适用于什么区域。 |
| `locale_name` | 字符串 | 区域名称。 |
| `description` | 字符串 | 节日或事件的文字说明。 |
| `transferred` | 布尔值 | 表示该节假日是否被调休或转移到其他日期庆祝。 |

## 8. 汇总说明

- 这套数据的预测粒度是 `date + store_nbr + family`。
- `sales` 是销量，不是销售额金额。
- `onpromotion` 不是布尔值，而是该商品大类下正在促销的商品数量。
- `type` 和 `cluster` 都可以视为类别特征。
- `transactions` 和销量通常强相关，但未来日期的真实 `transactions` 未必可直接获得，建模时要注意信息泄露。
- `dcoilwtico` 可能存在缺失值，常见做法是前向填充、插值或构造平滑特征。
- `holidays_events.csv` 不只包含法定节假日，还包含地方纪念日、转移假期、桥接假日、补班日和其他事件。
- 当 `locale = National` 时，通常作用于全国所有门店。
- 当 `locale = Regional` 时，通常需要按门店所在 `state` 匹配。
- 当 `locale = Local` 时，通常需要按门店所在 `city` 匹配。
- `Holiday` 表示正常节假日，`Additional` 表示额外假日，`Bridge` 表示桥接假日，`Transfer` 表示调休产生的转移日期，`Work Day` 表示补班日，`Event` 表示普通事件。
- `sample_submission.csv` 只是提交格式模板，最终提交时只需要 `id` 和 `sales` 两列。
